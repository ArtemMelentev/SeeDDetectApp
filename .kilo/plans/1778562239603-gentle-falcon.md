# План внедрения алгоритма v3 (paper+warp) в приложение SeeDDetect

## Цель
Внедрить обновлённую версию алгоритма из `AlgorithmVersions/VersionWithoutScan2/run_pipeline_v3.py` в приложение, заменив текущую интеграцию v2 в `segment_seeds_scan.py`, при этом:

- публичный контракт `segment_seeds_scan.analyze_image(...) -> dict` остаётся прежним (ключи, коды ошибок, формат `{"ok": ...}`);
- имена и набор артефактов остаются прежними:
  - `<stem>_all_mask.png`, `<stem>_all_overlay.jpg`, `<stem>_black_mask.png`, `<stem>_black_overlay.jpg`;
  - в `release_mode=True` пишется только `black_overlay` (как сейчас);
- Android/Chaquopy точка входа `android/app/src/main/python/android_bridge.py` не меняется.

## Контекст (что сейчас)
- `android_bridge.run_analysis_json` вызывает `segment_seeds_scan.analyze_image`.
- В `segment_seeds_scan.py` уже портирован v2 pipeline (см. комментарии и блок `# v2 helpers...`).
- Внутри `analyze_image` используется цепочка v2: `detect_paper_mask -> crop_to_paper(bbox) -> (upscale) -> normalize -> segment -> fixup`.
- `tests/test_segment_contract.py` проверяет стабильность payload и наличие артефактов.

## Что меняется в v3 относительно v2 (важное для интеграции)
- Детект бумаги стал более устойчивым и возвращает не только маску, но и 4 угла (quad) в `detect_info["quad"]`.
- Вместо bbox-кропа используется перспективное выпрямление: `warp_to_paper(img, paper_mask, quad)`.
- Это уменьшает протечки стола/фона в сегментацию и стабилизирует геометрию.

## Интеграционная стратегия
Минимальная по внешнему контракту замена: обновить реализацию v2-хелперов и блок обработки внутри `segment_seeds_scan.py` так, чтобы `analyze_image` работал по схеме v3.

Публичные API и структура возвращаемого словаря не расширяются.

## План изменений по файлам
### 1) `segment_seeds_scan.py`
1. Обновить комментарии и дефолты:
   - заменить ссылку `# v2 pipeline defaults ...run_pipeline_v2.py` на v3 (`...run_pipeline_v3.py`).

2. Заменить/обновить «v2 helpers» на «v3 helpers» (портировать вычислительную часть из `run_pipeline_v3.py`, без GUI/CLI/батч-раннера):
   - общие хелперы (можно переиспользовать существующие, но привести к фактам v3):
     - `_odd`, `_fill_holes`, `_component_bbox` (и при необходимости `_largest_component`).
   - геометрия quad и линии:
     - `_order_quad`, `_quad_from_contour`, `_line_from_points`, `_intersect_lines`.
   - v3-детект бумаги (внутренние подфункции):
     - `_score_paper_component`;
     - `_quad_from_paper_edges`;
     - `_moving_average_1d`, `_first_paper_run`;
     - `_refine_quad_by_border_color`;
     - `_cyan_field_mask`, `_table_mask`;
     - заменить текущую `_detect_paper_mask(...)` на v3-версию.
       - Важно: обеспечить, что возвращаемый `info` всегда содержит ключ `"quad"` (включая fallback full-frame).
   - добавить `_warp_to_paper(...)` (порт `warp_to_paper` из v3).

3. Обновить pipeline в `analyze_image`:
   - было (v2): `paper_full -> crop_to_paper(bbox) -> ...`.
   - станет (v3):
     1. `paper_full, detect_info = _detect_paper_mask(prepared_img)`
     2. `quad = np.array(detect_info["quad"], dtype=np.float32)`
     3. `crop_img, crop_paper, _warp_info = _warp_to_paper(prepared_img, paper_full, quad)`
     4. (опционально) upscale до `target_short` как сейчас
     5. `normalized = _normalize_scan(crop_img, crop_paper, do_shading=...)`
     6. `normalized = _white_balance_to_paper(normalized, crop_paper)`
     7. сегментация/фиксация как сейчас (`_segment_all_seeds_*`, `_segment_black_seeds_*`, `_fixup_masks`)
   - сохранить текущую логику формирования метрик/процентов и записи артефактов.

4. Совместимость и устойчивость:
   - оставить существующую обработку исключений и коды ошибок.
   - добавить локальный fallback вокруг `_warp_to_paper`: если OpenCV бросает исключение или получился слишком маленький `out_w/out_h`, использовать прежний bbox-кроп как запасной вариант (через `_component_bbox(paper_full)` и срезы). Это повышает живучесть на неожиданных входах без изменения контракта.

5. Не трогать:
   - `android/app/src/main/python/android_bridge.py`.
   - тестовый контракт payload (не добавлять новые ключи в верхний уровень результата).

### 2) Тесты
- Ожидается, что текущие тесты `tests/test_segment_contract.py` пройдут без изменений.
- Если какой-то синтетический кейс начнёт падать из-за агрессивного shrink/warp, разрешён минимальный фикс:
  - смягчить fallback (например, если площадь `paper_full` слишком мала или quad деградирует) и возвращаться к full-frame/ bbox-кропу.

## Проверка (выполнить после внедрения)
1. Прогнать unit-тесты:
   - `python -m unittest -q`
2. Быстрая ручная проверка на 2–3 реальных фото (если доступны):
   - убедиться, что создаются все артефакты;
   - визуально проверить `*_black_overlay.jpg` на адекватность;
   - сравнить протечки фона/стола с v2 (ожидается меньше протечек).

## Критерии готовности
- `tests/test_segment_contract.py` проходит.
- `analyze_image` возвращает прежние ключи и пишет прежние артефакты.
- Детект бумаги и ROI для сегментации работает через warp (v3), а не bbox (v2).
- В режиме `release_mode=True` создаётся только `black_overlay`, без масок.
