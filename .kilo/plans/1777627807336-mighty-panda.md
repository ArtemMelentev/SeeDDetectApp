# План: сохранять фото с кнопки «Сделать снимок» в галерею

## Цель
- После съемки через экран камеры фото должно автоматически попадать в системную галерею Android.
- Текущий поток анализа (возврат `XFile` и запуск `analyzeImage`) должен остаться рабочим.

## Что проверено в коде
- Кнопка `Сделать снимок` запускает `CameraCaptureScreen` из `lib/main.dart` (`_captureFromCamera`).
- `CameraCaptureScreen` делает `controller.takePicture()` и возвращает `XFile` через `Navigator.pop(file)` (`lib/camera_capture_screen.dart`).
- Сейчас сохранения в MediaStore/галерею нет: снимок только используется дальше для анализа.
- Платформенный канал уже существует: `seed_detect/analyzer` в `lib/main.dart` и `android/app/src/main/kotlin/com/seeddetect/app/MainActivity.kt`.
- Проект Android-only, `minSdk 29` (`android/app/build.gradle`), значит можно использовать scoped storage + `MediaStore` без legacy-пути.

## План изменений

### 1) Flutter: добавить вызов сохранения в галерею после съемки
Файл: `lib/main.dart`
- Добавить метод (например, `_saveCapturedPhotoToGallery`) для вызова платформенного метода через существующий `_channel`.
- Передавать путь сделанного снимка (`photo.path`) и опционально имя файла.
- В `_captureFromCamera` после получения `photo != null`:
  - вызвать сохранение в галерею в `try/catch`;
  - не ломать основной сценарий при ошибке сохранения (анализ должен запускаться в любом случае);
  - при необходимости показывать `SnackBar` о результате (успех/ошибка) без блокировки анализа.

### 2) Android: реализовать сохранение файла в MediaStore
Файл: `android/app/src/main/kotlin/com/seeddetect/app/MainActivity.kt`
- Расширить `MethodChannel`-обработчик новым методом, например `saveImageToGallery`.
- Реализовать функцию сохранения:
  - валидация входного пути (файл существует, читаем);
  - определение `mimeType` по расширению (fallback `image/jpeg`);
  - создание записи через `contentResolver.insert(MediaStore.Images.Media.EXTERNAL_CONTENT_URI, ContentValues(...))`;
  - запись байтов в `OutputStream` по `contentUri`;
  - для API 29+ использовать `IS_PENDING=1` во время записи и сбрасывать в `0` после успеха;
  - относительный путь `Pictures/SeedDetect` (или аналогичный каталог внутри `Pictures`), чтобы пользователь видел снимки в галерее.
- Возвращать в Flutter структурированный результат (`ok`, `savedUri`/`error`) и нормальную обработку исключений.

### 3) Обновить локализацию пользовательских сообщений в UI
Файл: `lib/main.dart`
- Добавить короткие русскоязычные сообщения для `SnackBar`:
  - успех: снимок сохранен в галерею;
  - ошибка: снимок сделан, но сохранить в галерею не удалось.
- Сообщения не должны мешать текущим сообщениям анализа.

### 4) Регрессия и тесты
- Автотесты:
  - проверить, что текущие widget/integration тесты проходят без изменений в существующем сценарии анализа;
  - при необходимости поправить мок платформенного канала, если новый метод вызовется в тестовом потоке.
- Ручная проверка на Android:
  - открыть `Сделать снимок`, сделать фото;
  - убедиться, что анализ запускается как раньше;
  - открыть системную галерею и проверить наличие нового фото в `Pictures/SeedDetect`;
  - повторить несколько снимков и проверить отсутствие падений/зависаний;
  - проверить сценарий ошибки сохранения (например, искусственно с невалидным путем через отладку) — анализ не должен ломаться.

## Технические заметки и ограничения
- Дополнительные runtime-permission для записи на Android 10+ не требуются при записи через `MediaStore`.
- Поскольку `minSdk 29`, ветка для старого `WRITE_EXTERNAL_STORAGE` не нужна.
- Сохраняем реализацию внутри текущего платформенного канала, без добавления сторонних Flutter-плагинов.

## Критерии готовности
- Фото, сделанное через `Сделать снимок`, появляется в системной галерее.
- Текущий UX анализа не деградирует: возврат из камеры и запуск обработки работают как раньше.
- При ошибке сохранения пользователь получает понятное уведомление, приложение не падает.
