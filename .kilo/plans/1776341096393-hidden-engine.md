# План: понятные метрики и интерактивный просмотр изображений

## Цель
Сделать финальный вывод результата анализа более понятным для пользователя:
1) добавить обязательную итоговую метрику в процентах для черных семечек относительно всех семечек;
2) явно показать оба базовых отношения площадей;
3) дать возможность открыть каждый артефакт и взаимодействовать с изображением (zoom/pan).

## Текущее состояние (по коду)
- Python-ядро (`segment_seeds_scan.py`) уже считает `all_seed_area_px`, `black_seed_area_px`, `image_area_px`, но не отдает процентные отношения в полезном для UI виде.
- Flutter-экран (`lib/main.dart`) показывает сырые метрики (count/area), а артефакты отображаются только как статичные превью.
- Контрактный тест (`tests/test_segment_contract.py`) проверяет базовые поля, но не проверяет процентные метрики.

## Принятые определения метрик
Чтобы закрыть формулировку задачи без двусмысленности, вводим три процента:

1. **Доля всех семечек в изображении, %**  
   `all_seed_ratio_pct = all_seed_area_px / image_area_px * 100`

2. **Доля черных семечек в изображении, %**  
   `black_seed_ratio_pct = black_seed_area_px / image_area_px * 100`

3. **Обязательная итоговая метрика, %** (черные относительно всех)  
   `black_to_all_seed_ratio_pct = black_seed_area_px / all_seed_area_px * 100`

Дополнительно в UI рядом с итоговой метрикой показываются оба исходных значения (`black_seed_area_px` и `all_seed_area_px`) и их доли в изображении.

## План реализации

### Этап 1. Расширить Python payload новыми метриками
Файл: `segment_seeds_scan.py`

1. После получения `all_stats` и `blk_stats` в `analyze_image(...)` вычислить процентные поля.
2. Добавить безопасное деление (защита от деления на ноль, если `all_seed_area_px == 0` или `image_area_px == 0`).
3. Добавить в итоговый `dict` новые ключи:
   - `all_seed_ratio_pct`
   - `black_seed_ratio_pct`
   - `black_to_all_seed_ratio_pct`
4. Сохранить обратную совместимость: существующие поля не переименовывать и не удалять.

Ожидаемый результат: Flutter получает готовые, однозначные процентные метрики без дополнительной бизнес-логики в UI.

### Этап 2. Перестроить блок метрик в Flutter для читабельности
Файл: `lib/main.dart`

1. В `_buildMetrics(...)` добавить явный раздел с финальной метрикой:
   - «Черные / все семечки: XX.XX%» (главная строка).
2. Под этой строкой показать «оба значения»:
   - `black_seed_area_px` (px)
   - `all_seed_area_px` (px)
   - и их доли в изображении (`black_seed_ratio_pct`, `all_seed_ratio_pct`).
3. Текущие count-метрики оставить, но визуально второстепенными.
4. Добавить форматирование процентов до 2 знаков после запятой и аккуратные подписи на русском.

Ожидаемый результат: пользователь сразу видит ключевую бизнес-метрику и ее исходные составляющие.

### Этап 3. Сделать артефакты кликабельными с zoom/pan
Файл: `lib/main.dart`

1. Обернуть превью в `InkWell`/`GestureDetector`.
2. По нажатию открывать полноэкранный просмотр (`Dialog`/новый route) с:
   - `InteractiveViewer` для pinch-to-zoom и pan;
   - `Image.file` внутри viewer;
   - кнопкой закрытия.
3. Добавить минимальную подсказку на превью («Нажмите для увеличения»).
4. Обработать edge-case: если файла нет, показывать понятное сообщение и не открывать viewer.

Ожидаемый результат: каждое изображение можно открыть и интерактивно исследовать.

### Этап 4. Обновить проверки и документацию
Файлы: `tests/test_segment_contract.py`, `README.md`

1. В контрактный Python-тест добавить проверки наличия новых полей и корректного диапазона (`0..100`).
2. Обновить раздел контракта в `README.md`, перечислив новые процентные метрики.
3. Выполнить локальные проверки:
   - Python: `py -m unittest -v`
   - Flutter: `flutter test`

Ожидаемый результат: изменения формально зафиксированы и защищены тестами.

## Критерии приемки (Definition of Done)
- В payload `analyze_image(...)` присутствуют три новых процента: `all_seed_ratio_pct`, `black_seed_ratio_pct`, `black_to_all_seed_ratio_pct`.
- В UI в конце блока метрик явно отображается обязательная метрика «черные / все семечки, %».
- В UI отдельно отображены оба значения площадей (black/all в px) и их доли в изображении.
- Любой артефакт (`all_mask`, `all_overlay`, `black_mask`, `black_overlay`) открывается по нажатию в интерактивном режиме с приближением/отдалением.
- Тесты контракта проходят с новыми полями, README отражает актуальный контракт.

## Риски и обработка
- Если `all_seed_area_px == 0`, итоговый процент должен быть `0.0`, без падения и `NaN`.
- Большие изображения в fullscreen могут быть тяжелыми: использовать существующий безопасный pre-resize для обработанных артефактов и не дублировать битмапы без необходимости.
- Не ломать текущие ключи payload, чтобы не нарушить совместимость native-bridge/UI.
