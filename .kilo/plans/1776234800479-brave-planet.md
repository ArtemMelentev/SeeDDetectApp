# План: запуск SeedDetectApp в Android-эмуляторе (Windows + VS Code)

## Цель (Definition of Done)
- Проект открывается в VS Code, зависимости ставятся без ошибок.
- Android-эмулятор (API 29+) запускается.
- Команда `flutter run` собирает и устанавливает приложение на эмулятор.
- В приложении можно выбрать изображение (gallery или files), обработка выполняется локально, отображаются метрики и 2–4 артефакта (mask/overlay).

## 0) Что уже готово в этом репозитории (текущий статус кода)
Это уже инициализированный Flutter-проект с подключенным Chaquopy и мостом Flutter↔Kotlin↔Python.

- Flutter UI: `lib/main.dart`
  - 2 кнопки выбора файла (gallery / files)
  - вызов native через `MethodChannel('seed_detect/analyzer')` метода `analyzeImage`
  - отображение метрик + картинок-артефактов по путям
- Android bridge (Kotlin): `android/app/src/main/kotlin/com/seeddetect/app/MainActivity.kt`
  - принимает `inputUri`
  - копирует файл в `cacheDir/seed_runs/run_.../input.ext`
  - запускает Chaquopy (Python 3.11) и вызывает `android_bridge.run_analysis_json`
- Python bridge: `android/app/src/main/python/android_bridge.py`
  - вызывает `segment_seeds_scan.analyze_image(input_path, output_dir)`
- Python ядро: `segment_seeds_scan.py`
  - есть библиотечная точка входа `analyze_image(...) -> dict` с контрактом `{ ok: true/false, metrics, artifacts }`
  - есть защита от OOM через downscale (параметры `max_pixels`, `max_side`)
- Chaquopy + зависимости Python уже прописаны: `android/app/build.gradle`
  - `numpy==1.26.4`
  - `opencv-python-headless==4.10.0.84`

Итого: твоя задача сейчас — не “написать приложение с нуля”, а поставить тулчейн и запустить уже подготовленный проект.

## 1) Что нужно установить (минимально необходимое)

### 1.1 Flutter SDK
Нужен Flutter (вместе с Dart).

Важно по версиям:
- В `pubspec.yaml` указано `sdk: ">=3.3.0 <4.0.0"`, значит нужен Flutter, который включает Dart 3.3+ (ориентир: Flutter 3.19+ / актуальный stable).

Рекомендуемый путь на Windows:
1) Скачать Flutter SDK (stable) с официального сайта.
2) Распаковать, например, в `C:\src\flutter`.
3) Добавить `C:\src\flutter\bin` в `PATH`.
4) Проверить:
   - `flutter --version`

### 1.2 Android Studio + Android SDK + Emulator
Нужен Android SDK, platform-tools (adb), build-tools и сам эмулятор.

Сделать в Android Studio:
1) Установить Android Studio.
2) В `SDK Manager` установить:
   - Android SDK Platform: минимум API 29 (Android 10), можно также самый свежий (для образов эмулятора)
   - SDK Tools: Android SDK Platform-Tools, Android SDK Build-Tools, Android SDK Command-line Tools (latest), Emulator
3) В `Device Manager` создать AVD (виртуальный девайс):
   - Любой Pixel
   - System image: x86_64 (для Windows) и API 29+ (можно 33/34)

Важно для эмулятора:
- В BIOS должна быть включена виртуализация (Intel VT-x / AMD-V).
- В Windows обычно нужен включенный `Windows Hypervisor Platform` (или Hyper-V). Если эмулятор не стартует — см. раздел “Проблемы”.

### 1.3 Java (JDK 17)
Проект использует Android Gradle Plugin 8.3.x и Kotlin, для них нужен JDK 17.

Варианты:
- Использовать JDK, который идет с Android Studio (Embedded JDK / JetBrains Runtime).
- Или установить Temurin/Oracle JDK 17 и выставить `JAVA_HOME` на него.

Проверка:
- `java -version` должен показывать 17.

### 1.4 Python 3.11 на машине (важно для Chaquopy)
Chaquopy ставит pip-зависимости во время сборки и для этого требует Python на машине сборки той же major.minor версии, что и в приложении.

В проекте указано: `chaquopy { defaultConfig { version "3.11" ... } }`.

Поэтому:
- Установить Python 3.11 (Windows installer) и убедиться, что доступен Python Launcher:
  - `py -3.11 --version`

Если Python Launcher не установлен, можно:
- поставить его вместе с Python (галка “py launcher”), или
- позже добавить `buildPython` в Gradle-конфиг (это уже шаг дебага при ошибке сборки).

## 3) Подготовка проекта (перед первым запуском)

В терминале VS Code, из корня репозитория `C:\SeeDDetectApp`:
1) Скачать Flutter-зависимости:
   - `flutter pub get`
2) Проверить окружение:
   - `flutter doctor -v`
3) Принять Android licenses (если попросит):
   - `flutter doctor --android-licenses`

Опционально (полезно, но не обязательно для первого запуска):
- Прогнать быстрые тесты UI:
  - `flutter test`
- Прогнать Python unit-тесты локально (понадобятся numpy+opencv в твоем Python):
  - `py -3.11 -m pip install -r requirements.txt`
  - `py -3.11 -m unittest -v`

## 4) Запуск Android-эмулятора

Вариант A (проще): через Android Studio
1) Android Studio → `Device Manager`
2) Выбрать созданный AVD → `Play` (Start)

Вариант B: через Flutter CLI
1) Посмотреть доступные эмуляторы:
   - `flutter emulators`
2) Запустить нужный:
   - `flutter emulators --launch <emulator_id>`

Проверка, что Flutter видит устройство:
- `flutter devices`

## 5) Первый запуск приложения на эмуляторе

1) Убедиться, что эмулятор запущен.
2) Из корня проекта выполнить:
   - `flutter run`

Что важно понимать про первый билд:
- Gradle будет скачивать Android зависимости.
- Chaquopy будет скачивать/собирать Python окружение и ставить `numpy`/`opencv`.
- Первый запуск может занять заметное время (минуты-десятки минут).

Ожидаемое поведение:
- Приложение установится на эмулятор.
- Откроется экран с двумя кнопками.

## 6) Как проверить сценарий “выбор файла → обработка → результат”

### 6.1 Добавить тестовую картинку в эмулятор
Чтобы было что выбрать:
- Самый простой способ: перетащить файл изображения (jpg/png) мышью на окно эмулятора.
  - Обычно файл окажется в `Download`.

Альтернатива через adb (если хочешь воспроизводимо):
- `adb devices`
- `adb push path\\to\\image.jpg /sdcard/Download/`

### 6.2 Прогон в приложении
1) Нажать `Select from files`.
2) Выбрать изображение из `Downloads`.
3) Дождаться завершения.
4) Убедиться, что:
   - показаны метрики (counts/area/size/ms)
   - показаны картинки (mask/overlay)

## 7) Типовые проблемы при сборке/запуске и что делать

### 7.1 `flutter doctor` ругается на Android toolchain
Симптомы:
- “Android SDK not found”
- “cmdline-tools component is missing”

Решение:
- Открыть Android Studio → SDK Manager → установить `Android SDK Command-line Tools (latest)`
- Убедиться, что SDK реально установлен (обычно `C:\Users\\<You>\\AppData\\Local\\Android\\Sdk`)
- Добавить переменную `ANDROID_SDK_ROOT` (или `ANDROID_HOME`) на этот путь

### 7.2 Не приняты лицензии
Симптом:
- Gradle/Flutter пишет про licenses.

Решение:
- `flutter doctor --android-licenses`

### 7.3 Java/JDK проблема (AGP требует 17)
Симптомы:
- “Unsupported class file major version”
- “Android Gradle plugin requires Java 17”

Решение:
- Поставить JDK 17 и выставить `JAVA_HOME`
- Перезапустить терминал/VS Code

### 7.4 Chaquopy не находит Python 3.11 на машине сборки
Симптом:
- Ошибка сборки в духе “buildPython not found” / “Python 3.11 required”

Решение:
- Установить Python 3.11 и проверить `py -3.11 --version`
- Если Python стоит, но Gradle его не видит — на этапе реализации можно будет явно указать `buildPython(...)` в `android/app/build.gradle`.

### 7.5 pip не может поставить `opencv-python-headless`
Симптом:
- “Could not find a version that satisfies the requirement opencv-python-headless…”

Что это значит:
- Для Android нужны специальные wheels. Не все версии/пакеты доступны для конкретной связки Chaquopy+Python.

Решение (план действий):
1) Сначала посмотреть текст ошибки: какие “from versions” он перечисляет (если перечисляет).
2) Ослабить/поменять версию `opencv-python-headless` на ту, которая реально доступна.
3) Если пакет в принципе недоступен — заменить на другой способ доставки OpenCV (например, другой пакет, который есть в репозитории Chaquopy).

Примечание: это нормальный риск для Android+Python. Если упрется именно в OpenCV, мы перейдем в “debug”-режим и подберем совместимую комбинацию версий.

### 7.6 Эмулятор не стартует или очень медленный
Частые причины:
- выключена виртуализация в BIOS
- не включены нужные компоненты Windows (Hypervisor Platform)

Решение:
- включить виртуализацию в BIOS
- включить Windows feature `Windows Hypervisor Platform`
- перезагрузить ПК

## 8) Итоговый минимальный чек-лист (без воды)
1) Установить Flutter SDK и добавить в `PATH`.
2) Установить Android Studio + SDK + Emulator.
3) Установить JDK 17 (`JAVA_HOME`).
4) Установить Python 3.11 (`py -3.11 --version`).
5) VS Code: поставить расширение Flutter.
6) В корне проекта: `flutter pub get`.
7) `flutter doctor -v` и `flutter doctor --android-licenses`.
8) Запустить AVD (Device Manager).
9) `flutter run`.
10) Закинуть картинку в эмулятор и прогнать `Select from files`.
